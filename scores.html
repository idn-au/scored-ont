Provenance

This ontology was developed as part of the Indigenous Data Network's Catalogue Profile (https://linked.data.gov.au/def/idncp) in 2022. The structures of the various Scores - FAIR, CARE etc. - reflect those principles' structures.

Description

This is an ontology designed to represent the scores that have been calculated for catalogues resources according to various principles, such as the CARE and FAIR principles. The ontology also provides properties to link Resources to scores.

Score c

IRI https://linked.data.gov.au/def/scores/Score
Is Defined By Scores Ontology
Description

A Score is a value, or set of values, for a catalogues Resource that has been calculated from metadata or data elements known for the resource in accordance with a scoring principles or metric system, such as the FAIR principles
Example 
        <http://example.com/resource/x>
            a dcat:Resource ;
            scores:hasScore <http://example.com/score/y> ;
        .

        <http://example.com/score/y>
            a scores:Score ;
        .

Object Properties

has score op

IRI https://linked.data.gov.au/def/scores/hasScore
Is Defined By Scores Ontology
Description

The subject Resource has a Score calculated for it from its metadata
Scope Note

As per Data Cube modelling, a Score, being a group of Observations, a qb:ObservationGroup, is linked to the thing that is scored (observed), the catalogues Resource, via a property of the Observation instance. This property then is an alternate linking mechanism and not the only one or even the primary one: it is Resource-centric, as opposed to Observations-centric.
Example 
        <http://example.com/resource/x>
            a dcat:Resource ;
            scores:hasScore <http://example.com/score/y> ;
        .

        <http://example.com/score/y>
            a scores:Score ;
        .
